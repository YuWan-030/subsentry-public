# SubSentry 前端

SubSentry 前端基于 React、Vite 和 Ant Design，提供资产总览、客户管理、客户详情、个人中心、节点集群、用户管理、系统设置、系统健康、日志中心和首次安装向导等页面。

## 开发启动

```powershell
cd frontend
npm install
npm run dev
```

只有当前端和后端不在同源地址时，才需要设置 `VITE_API_BASE_URL`。生产环境留空时，请求会自动走当前域名，例如 `/api/v1/auth/me`。

## 首次安装向导

全新后端数据库中没有管理员账号时，前端会自动跳转到 `/install`。安装向导会创建第一个管理员账号，并收集数据库、站点 URL 和 Webhook 配置。

如果安装向导保存了新的 MySQL 配置，或修改了 SQLite 文件路径，需要重启后端并刷新页面后继续。

## 生产构建

```powershell
npm run build
```

构建产物位于 `frontend/dist`。其中包含用于移动端添加到主屏幕的 manifest，生产部署时建议让前端入口、manifest 和静态资源都从同一个公开域名访问。

## Linux 一键安装

项目根目录的 `scripts/install-linux.sh` 会自动构建前端，并通过 Nginx 托管 `dist`。一键安装默认采用同源 API 请求，因此构建时会保持 `VITE_API_BASE_URL` 为空。

## 当前页面

- 资产总览：收入统计、状态概览、收入趋势。
- 客户管理：客户列表、筛选、批量操作、订阅链接、二维码、删除。
- 客户详情：资料编辑、续费、订阅链接、续费记录、审计记录、Webhook 测试。
- 个人中心：昵称、密码、OnAuth、Passkey。
- 节点集群：节点增删改、连接测试、订阅配置读取、节点探测。
- 用户管理：用户创建、禁用、昵称、密码、审计。
- 系统设置：通知模板、推送策略、站点展示、本地订阅服务。
- 系统健康：后端、数据库、节点探测、通知检查、自动任务状态。
- 日志中心：操作审计、通知发送、失败重试。
