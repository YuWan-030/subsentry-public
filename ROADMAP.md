# SubSentry Roadmap

## 2026-06-11 已落地

- 系统健康页已完成，管理员可查看后端、数据库、节点探测、通知检查和自动任务状态。
- 高频自动任务状态已改为轻量状态记录，不再把每 5 分钟一次的成功检查写入活动日志。
- 首次安装向导已完成，新部署可在 `/install` 配置管理员、数据库、站点 URL 和默认 Webhook。
- 客户流量倍率已完成，可支持五倍速率扣减等流量控制场景。
- PWA 和 iOS 主屏幕相关配置已补充，移动端可按网页 App 方式打开。
- Linux 一键安装脚本已完成，可安装依赖、构建前端、写入 `.env`、配置 systemd 和 Nginx 并启动面板。

这个文件记录 SubSentry 后续可以继续增强的方向，主要围绕“长期稳定运行、方便公网部署、方便开源协作”来规划。

## 优先级一：公网运行与开源准备

### 备份与恢复

- 数据库一键备份和下载。
- MySQL 备份说明。
- 配置导出和导入。
- 公网部署后，备份恢复是最实用的保险能力。

### 系统健康页 - 已完成

- 后端运行状态。
- 数据库连接状态。
- 节点探测状态。
- 最近一次通知检查时间。
- 最近一次自动任务执行结果。
- 目标是不用猜“任务到底有没有跑、系统到底哪里异常”。

### 通知检查日志 - 已调整为轻量状态

- 高频定时检查不再每次写入活动日志。
- 最近一次通知检查、节点探测和自动任务结果写入轻量任务状态。
- 通知发送明细仍保留在通知日志中，失败通知支持重试。
- 后续如需更完整的任务审计，可单独设计低频归档或按异常记录。

### 开源前脱敏工具

- 检查代码里是否存在 IP、密码、Webhook、Token 等敏感内容。
- 生成 `.env.example`。
- 自动检查 `frontend/dist`、数据库文件、日志文件是否误提交。
- 这是上传 GitHub 前的重要安全准备。

## 优先级二：部署体验与数据流转

### 安装向导 - 已完成

- 首次启动时设置管理员账号。
- 配置数据库。
- 配置站点 URL。
- 配置 Webhook。
- 降低其他用户部署 SubSentry 的门槛。

### Linux 一键安装 - 已完成

- 一条命令安装系统依赖、Node.js、Python venv、后端依赖。
- 自动构建前端静态文件。
- 自动写入 `.env` 默认配置。
- 自动创建 systemd 服务和 Nginx 站点。
- 默认 SQLite，安装后进入网页安装向导创建管理员。

### 客户导入与导出

- Excel/CSV 导入客户。
- 导出客户列表。
- 导出续费记录。
- 导出收入流水。
- 对运营类系统很实用，也方便数据迁移。

### 文档与截图

- README 增加界面截图。
- 增加部署示意图。
- 增加宝塔部署教程。
- 增加常见问题。
- 开源项目里，截图和部署图能让用户更快理解项目价值。

## 优先级三：团队协作与安全兜底

### 更细的权限

- 管理员。
- 客户经理。
- 只读账号。
- 财务账号。
- 当前 `admin/user` 已经够用，后续开源后可以按多人协作场景继续细化。

### 操作二次确认与回收站

- 删除客户前二次确认。
- 删除后保留一段时间。
- 支持误删恢复。
- 这类能力比很多新功能更能提升日常使用安全感。

### 仪表盘缓存 - 已完成

- 首页统计做短缓存，例如 10-30 秒。
- 移动端和 Safari 会更稳。
- 节点数据和财务数据可以分层刷新。

## 推荐实施顺序

1. 备份恢复。
2. 任务执行日志。
3. 系统健康页。
4. 脱敏工具和 `.env.example`。
5. 客户导入/导出。
6. README 截图、部署图和常见问题。
7. 客户导入/导出。
8. 更细权限、回收站。
